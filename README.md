# AI Swing — 強化学習でブランコを漕ぐ

![cold-start hero](hero_cold_start.png)

PPO (Proximal Policy Optimization) で学習したエージェントが、**完全に静止した状態からブランコを漕ぎ出し、約 260 秒で振幅 ±40° に到達**するまでの実装です。身体モデルは「ブランコの座面に座る人間」を 3 質点 (ブランコ棒, 胴体, 頭部) の連結剛体としてモデル化しており、エージェントは腰のトルクだけで身体を動かします。

## 成果サマリ

| 条件 | 最大振幅 \|x\| | 到達時間 |
|---|---|---|
| 完全静止スタート (x=0, dx=0, φ=0, dφ=0) | **~40°** | ~260 s でプラトー |
| 初期振幅 15° スタート | ~40° | ~160 s でプラトー |

- 腰トルク上限: ±500 Nm
- 頭部は胴体に剛結合 (首関節の剛性を上げて実質ロック)
- 物理タイムステップ: 1 ms, AI 判断周期: 20 ms

## 物理モデル

3 自由度 $(x, \phi, \alpha)$ の結合系:

- $x$: ブランコ棒の鉛直からの角度 (人体とブランコ棒を同一角度で一体運動)
- $\phi$: 胴体の鉛直からの絶対角度
- $\alpha$: 頭部の鉛直からの絶対角度

質量分布は `cN=[2XS,XS,Sv,Sr,M,L]` のブランコサイズ選択と、体重 50 kg/身長 1.58 m の人体セグメント (胴体 54.5%, 頭部 8.1%, 上腿 24.6%, 下腿 12.8%) から組み立てています。

運動方程式は慣性行列 × 加速度 = 外力+結合項という 3×3 連立を立て、各ステップでクラメルの公式で直接解いてから RK4 を回します。腰関節・首関節はそれぞれバネ＋ダンパでモデル化し、重力による胴体の倒立不安定を越える剛性を持たせています (後述)。

## 強化学習の定式化

### 行動空間
```python
action_space = Box(low=-500.0, high=500.0, shape=(1,))  # 腰トルク [Nm]
```
頭部トルクは動作中常に 0。アクションは `u_body` のみの 1 次元。

### 観測空間
```python
obs = [phi, d_phi, alpha, d_alpha, x, z]   # 6 次元
```

### 報酬
```python
h = 1.0 - math.cos(self.sw.x)     # ブランコの「持ち上がり高さ」(無次元)
reward = h * h * 1000.0
```
振幅の 2 乗で効くので、大振幅ほど比例以上に報酬が伸びる。

### 終了判定
```python
terminated = (|x| > π)                       # 1 回転
          or (|phi - x| > 60°)               # 胴体が座面に対して倒れすぎ
          or (|alpha - phi| > 60°)           # 頭部が胴体に対して倒れすぎ (実質起きない)
truncated  = (t > 200 s)
```

### 初期条件 (バイアス付きランダム)
```python
if random.random() < 0.4:
    scale = 0.0                              # 40%: 完全静止スタート
else:
    scale = random.uniform(0.0, 1.0)         # 60%: 振幅 0〜15° の範囲でランダム
```
静止スタートを 4 割に増やすことで、エージェントが「種火を作る」局面の経験を十分積めるようにしています。

### PPO ハイパーパラメータ
```python
PPO("MlpPolicy", env, learning_rate=3e-4, ent_coef=0.01)
model.learn(total_timesteps=3_000_000)
```

## 学習に至るまでに工夫した点

ここからが本題です。単純な定式化ではうまく行かず、何度も行き詰まりながら物理と報酬と学習設定を噛み合わせていきました。

### 1. 報酬設計の変遷

| 世代 | 報酬 | 問題 | 対策 |
|---|---|---|---|
| v1 | **エネルギー増分** `(E_new - E_old) * 60` | エネルギーが下がる局面 (振幅が一時的に縮む) で負の報酬、しかし正味は増えていても step ごとにノイジーで学習が進まない | 累積的な振幅指標に変える |
| v2 | **振幅の関数** `(1 - cos(x))` (位置エネルギー比例) | 振幅自体は伸びたが、停留点で常に低報酬になるので「常時動いていたい」圧が弱く、平衡点で大きく振れる単一モード解に収束しない。また第 2 モード (胴体が速く振動する解) で報酬ハッキング | 2 乗にしてピーク差を強調 |
| v3 (採用) | **2 乗化** `(1 - cos(x))^2 * 1000` | 大振幅ほど指数的に得、小振幅は無視できる程度の報酬。エージェントが「とにかく振幅を稼ぐ」方向に純粋に最適化 | — |

> 「報酬ハッキング」について: 中間実験で周期 0.75 s の高速振動が混入し、計算した周波数が `1/T = 1.3 Hz` 付近 (ブランコの固有振動 0.38 Hz の約 3.5 倍) になることがありました。FFT で調べた結果、これは胴体-ブランコ結合系の**第 2 正規モード**で、胴体を高速に揺らすだけで `1-cos(x)` が少し増える状態を見つけていました。2 乗化＋次項のダンパ調整で抑制。

### 2. 人体と座面の結合剛性・減衰

```python
k_waist = 144.0  # [Nm/rad]
c_waist = 10.0   # [Nm·s/rad]
k_neck  = 2000.0 # [Nm/rad]
c_neck  = 50.0   # [Nm·s/rad]
```

工夫ポイント:

- **`k_waist = 144`** の根拠: 胴体は座面に対して倒立振子なので、重力だけで約 113 Nm/rad の不安定化トルクを受ける。これを上回る剛性 (113 + 余裕 31 = 144 Nm/rad) にして、エージェントが「バランスを取る」作業に貴重なトルク予算を吸われずに、漕ぎ出しに集中できるようにした。これで胴体の固有周期は約 2.65 s。
- **`c_waist = 10`** の根拠: 過去に `c=30` にしたら消散が大きすぎて、入力電力 ~3.5 W < 減衰電力 ~7.5 W となり振幅が育たなかった。パワーバランスを確認して 10 に下げ、漕ぎを許す一方で第 2 モードを抑える程度に調整。臨界減衰比 $\zeta \simeq 0.13$ 相当。
- **首関節は実質ロック**: 当初は頭部トルクも行動に入れて自由度 2 の漕ぎを学習させようとしたが、`k_neck=5` など弱い値では学習序盤で頭が 60° 以上倒れ、`alpha - phi` の終了条件で 0.5 s でエピソードが終わっていた。`k_neck=2000, c_neck=50` にして頭部を胴体に剛結合相当にし、行動空間も 1 次元にして安定化。

### 3. トルクのローパスフィルター

```python
# 1次 IIR: 時定数 tau_u = 0.1 s
alpha_f = h / self.tau_u
self.u_body_filt += (u_body - self.u_body_filt) * alpha_f
u_body = self.u_body_filt
```

初期の学習結果で胴体が極端に速く (~20 Hz) 振動していた問題への対策。エージェントは 50 Hz で判断しているので、フィルタを通さないと 25 Hz ナイキストまでの帯域を全部使って細かく震える解に落ち込む。$\tau_u = 0.1\ \text{s}$ (カットオフ ~1.6 Hz) で人間らしいゆっくりした漕ぎに矯正した。

> 最初は $\tau_u=0.3\ \text{s}$ から始めたが、そこまで絞ると振幅が半減したので 0.1 s に緩めた。

### 4. エピソード長 (`t > 200 s`) と「テストが短すぎた」問題

- 学習エピソードは 200 秒 (= 10,000 env steps) まで。物理の固有周期が ~2.65 s なので、漕ぎ出しに 100 回以上のスイングを消費できる。
- 学習中盤で一時的に `ep_rew_mean` が伸び悩み、テスト走行 (40 s = 2000 steps) では ±15° 止まりになる現象があった。これは**テストが短すぎただけ**で、同じモデルを 600 秒走らせると実は ±49° まで育つことが判明。以後、テスト走行は 200 秒以上取るようにしている。

### 5. 初期条件のランダム化 (静止スタート 40%)

学習時に `x_init = -15°` 固定で訓練すると、**鉛直静止からは漕ぎ出せないエージェント**が得られた (`x = 0` から始めると 600 秒経っても ±1.5° しか揺れない)。これは訓練分布に「静止状態」が一度も含まれていないため。

対策として `reset()` で

- 40% の確率で完全静止 (`scale = 0`)
- 60% の確率で `scale ~ U(0, 1)` で初期振幅スケーリング

と振り分け、静止スタートを他の任意の初期振幅より高い頻度で見せるようにした。3M timesteps 再学習の結果、静止から自発的に漕ぎ上がる方策が得られた (本 README トップの画像の結果)。

### 6. 対称な終了条件

`|phi - delta_phi_0| > 65°` という「胴体の絶対角制限」は、ブランコが傾いている状態では左右非対称に作用してしまい (片側だけ早く terminate される)、ハックしやすい不均衡を生んでいた。`|phi - x| > 60°` に変えて座面相対角で終了判定することで左右対称にした。

### 7. 探索強化 (`ent_coef = 0.01`)

v3 報酬で学習しても ±15° の局所最適に居座ってしまう問題があった (トルクは ±7 Nm しか使わない、いわば「諦めて小さく揺れる」解)。

PPO の `ent_coef` (エントロピーボーナス) をデフォルト 0 から 0.01 に上げて方策分布の乱雑さを促し、より大きなトルクを試行させるようにしたところ、脱出して ±40° 以上まで漕ぎ上がる解が得られた。

### 8. 学習ステップ数を 1M → 3M に拡大

漕ぎ出しの「種火作り」局面が長い (最初の 60 秒で振幅はまだ 6° 程度) ため、1 エピソードあたりの経験が厚い。ひとつの方策を十分にクレジット割り当てできるよう 3M timesteps まで増やした。fps ≈ 1640 で約 30 分で完了。
